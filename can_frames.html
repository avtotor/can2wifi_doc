CAN фреймы

Структура фреймов, OBD-II PID, работа с SavvyCAN

Структура CAN 2.0 фрейма

Standard Frame (11-bit ID)

Диапазон ID0x000 — 0x7FF
ID в прошивкеextended = false
ПрименениеOBD-II, большинство легковых авто
Максимум ID2048 уникальных

Extended Frame (29-bit ID)

Диапазон ID0x00000000 — 0x1FFFFFFF
ID в прошивкеextended = true (бит 31 в GVRET)
ПрименениеJ1939 (грузовики), диагностика
Максимум ID~537 миллионов

Структура CAN_FRAME в прошивке

// esp32_can_idf.h

typedef union {
    uint8_t bytes[8];      // доступ побайтно
} CAN_DATA;

typedef struct {
    uint32_t id;           // 11-bit или 29-bit CAN ID
    uint32_t timestamp;    // микросекунды с момента старта
    bool     extended;     // true = 29-bit extended ID
    uint8_t  length;       // DLC: 0-8 байт данных
    uint8_t  rtr;          // Remote Transmission Request
    CAN_DATA data;         // 8 байт данных
} CAN_FRAME;

OBD-II диагностика через CAN

Прошивка не содержит встроенных OBD-II PID! Она является прозрачным шлюзом — все фреймы передаются в SavvyCAN как есть. Но вы можете отправлять OBD-II запросы из SavvyCAN вручную.

Как работает OBD-II запрос/ответ

Формат OBD-II запроса

CAN ID: 0x7DF (broadcast) или 0x7E0 (ECU #1)
DLC:    8 (всегда 8 байт, неиспользуемые = 0x00)

Байт 0: Кол-во значимых байт (обычно 02)
Байт 1: Режим (Service)
Байт 2: PID
Байт 3-7: 0x00 (padding)

Формат OBD-II ответа

CAN ID: 0x7E8 (ответ от ECU #1 = 0x7E0 + 8)
DLC:    8

Байт 0: Кол-во значимых байт
Байт 1: Режим + 0x40 (например 0x01 → 0x41)
Байт 2: PID
Байт 3+: Данные (зависит от PID)

Пример: запрос оборотов (RPM)

ЗАПРОС (SavvyCAN → CAN шина):
  ID: 0x7DF  DLC: 8  Data: 02 01 0C 00 00 00 00 00
                                   │  │  │
                                   │  │  └─ PID 0x0C (RPM)
                                   │  └──── Service 01 (текущие данные)
                                   └─────── 2 значимых байта

ОТВЕТ (ECU → SavvyCAN):
  ID: 0x7E8  DLC: 8  Data: 04 41 0C 1A F8 00 00 00
                                   │  │  │  │  │
                                   │  │  │  │  └─ B = 0xF8 = 248
                                   │  │  │  └──── A = 0x1A = 26
                                   │  │  └─────── PID 0x0C
                                   │  └────────── 0x41 = Service 01 + 0x40
                                   └───────────── 4 значимых байта

  RPM = (A * 256 + B) / 4 = (26 * 256 + 248) / 4 = 1718 об/мин

Приоритет: в CAN шине побеждает фрейм с меньшим ID (арбитраж). Поэтому критические системы (тормоза, мотор) используют низкие ID, а диагностика OBD-II — высокие.

4IP: 192.168.4.1, Port: 23

Отправка фрейма из SavvyCAN

ШагДействие
1SavvyCAN → Send Frames
2ID: 7DF, Bus: 0, Len: 8
3Data: 02 01 0C 00 00 00 00 00 (RPM запрос)
4Нажать Send
5Искать ответ с ID 0x7E8 в списке фреймов
Осторожно с отправкой! В режиме NORMAL прошивка может отправлять фреймы в CAN шину автомобиля. Неправильные фреймы могут вызвать неожиданное поведение. Для безопасного мониторинга используйте Listen-Only режим (настраивается через GVRET команду SETUP_CANBUS).

Фильтрация фреймов

В текущей прошивке TWAI настроен на приём всех фреймов (TWAI_FILTER_CONFIG_ACCEPT_ALL()). Фильтрация не используется — все фреймы передаются в SavvyCAN, где можно фильтровать программно.

Аппаратная фильтрация TWAI (если включить)

ESP32 TWAI поддерживает один аппаратный фильтр:

twai_filter_config_t f_config = {
    .acceptance_code = 0x7E000000,  // ID для сравнения (сдвинут влево)
    .acceptance_mask = 0x001FFFFF,  // 0 = проверять бит, 1 = игнорировать
    .single_filter   = true         // один фильтр на все фреймы
};

Правило: (входящий_ID & ~маска) == (код & ~маска) → принять

В нашей прошивке: acceptance_mask = 0xFFFFFFFF (все биты игнорируются → принимать всё)

Счётчики ошибок CAN

Каждый узел CAN ведёт два счётчика (доступны через twai_get_status_info):

TEC (Transmit Error Counter):
  0-95    : Error Active  → нормальная работа
  96-127  : Warning        → начались проблемы
  128-255 : Error Passive  → может только слушать
  >255    : Bus-Off        → отключён от шины!

REC (Receive Error Counter):
  Аналогично TEC, но для приёма.

При Bus-Off TWAI можно восстановить: twai_initiate_recovery().
Текущее состояние: twai_get_status_info(&status) → status.state
  TWAI_STATE_RUNNING / TWAI_STATE_BUS_OFF / TWAI_STATE_RECOVERING
